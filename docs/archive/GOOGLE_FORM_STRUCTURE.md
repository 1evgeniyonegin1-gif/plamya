# Структура Google-формы для онбординга партнёров

## Как создать форму

1. Перейди на https://forms.google.com/
2. Создай новую форму
3. Добавь разделы и вопросы по структуре ниже
4. Включи загрузку файлов (для JSON-ключа)
5. Настрой уведомления на email

---

## Структура формы

### Заголовок формы
```
Заявка на подключение AI-бота NL International

Заполни форму, чтобы получить своего AI-куратора для работы с клиентами.
Время заполнения: ~10 минут

Перед заполнением убедись, что ты:
1. Создал бота в @BotFather (инструкция: [ссылка])
2. Создал группу в Telegram
3. Зарегистрировался в Yandex Cloud (инструкция: [ссылка])
```

---

### Раздел 1: Основная информация

**Вопрос 1.1**
- Тип: Короткий текст
- Вопрос: `Твоё имя`
- Описание: `Как бот будет представляться клиентам. Например: "Анна", "Михаил"`
- Обязательный: Да

**Вопрос 1.2**
- Тип: Короткий текст
- Вопрос: `Твой Telegram для связи с клиентами`
- Описание: `В формате @username. Клиенты смогут написать тебе напрямую.`
- Обязательный: Да
- Валидация: Регулярное выражение `^@[a-zA-Z0-9_]+$`

**Вопрос 1.3**
- Тип: Короткий текст
- Вопрос: `Телефон (опционально)`
- Описание: `Если хочешь, чтобы клиенты могли позвонить`
- Обязательный: Нет

---

### Раздел 2: Telegram-бот

**Вопрос 2.1**
- Тип: Длинный текст
- Вопрос: `Токен бота`
- Описание: `Получи в @BotFather после создания бота. Выглядит как: 7123456789:AAHx1234567890abcdefghijklmnop`
- Обязательный: Да

**Вопрос 2.2**
- Тип: Короткий текст
- Вопрос: `Username бота`
- Описание: `В формате @username_bot`
- Обязательный: Да
- Валидация: Регулярное выражение `^@[a-zA-Z0-9_]+bot$`

---

### Раздел 3: Telegram-группа

**Вопрос 3.1**
- Тип: Короткий текст
- Вопрос: `ID группы`
- Описание: `Начинается с -100... Получи через @getidsbot`
- Обязательный: Да
- Валидация: Регулярное выражение `^-100[0-9]+$`

**Вопрос 3.2**
- Тип: Короткий текст
- Вопрос: `Название группы`
- Описание: `Для идентификации`
- Обязательный: Да

**Вопрос 3.3**
- Тип: Множественный выбор (чекбоксы)
- Вопрос: `Подтверди:`
- Варианты:
  - `Бот добавлен в группу`
  - `Бот назначен администратором`
  - `У бота есть права на публикацию сообщений`
- Обязательный: Да (все галочки)

---

### Раздел 4: Yandex Cloud

**Вопрос 4.1**
- Тип: Короткий текст
- Вопрос: `Folder ID`
- Описание: `Из консоли Yandex Cloud. Пример: b1g8skpblkos03malf3s`
- Обязательный: Да

**Вопрос 4.2**
- Тип: Короткий текст
- Вопрос: `Service Account ID`
- Описание: `ID сервисного аккаунта. Пример: aje76dc7i20078podfrc`
- Обязательный: Да

**Вопрос 4.3**
- Тип: Короткий текст
- Вопрос: `Key ID`
- Описание: `ID ключа из JSON-файла (поле "id"). Пример: ajensd96tl0d2q9fqmp9`
- Обязательный: Да

**Вопрос 4.4**
- Тип: Загрузка файла
- Вопрос: `JSON-файл с ключом`
- Описание: `Скачанный файл при создании авторизованного ключа в Yandex Cloud`
- Обязательный: Да
- Ограничения: только .json, максимум 1 файл, максимум 1 MB

### Раздел 5: NL International

**Вопрос 5.1**
- Тип: Короткий текст
- Вопрос: `Твой ID партнёра NL`
- Описание: `Числовой ID из личного кабинета NL`

**Вопрос 5.2**

- Вопрос: `Ссылка на магазин с твоим ID`

**Вопрос 5.3**

- Вопрос: `Ссылка на регистрацию партнёра`
- Описание: `Пример: https://nlstar.com/ref/abc123/`
- Обязательный: Да
- Валидация: URL

---

### Раздел 6: Профиль бота

**Вопрос 6.1**
- Тип: Множественный выбор (радио)
- Вопрос: `Твоя основная специализация`
- Описание: `Бот будет делать акцент на выбранной теме`
- Варианты:
  - `Похудение / контроль веса`
  - `Энергия / витамины / здоровье`
  - `Бизнес / заработок / партнёрство`
  - `Красота / уход за собой`
  - `Всё понемногу (универсальный)`
- Обязательный: Да

**Вопрос 6.2**
- Тип: Длинный текст
- Вопрос: `Твоя история`
- Описание: `2-3 предложения: почему ты в NL, какие результаты у тебя или твоих клиентов. Бот будет использовать это в общении.`
- Обязательный: Да

**Вопрос 6.3**
- Тип: Множественный выбор (радио)
- Вопрос: `Стиль общения бота`
- Варианты:
  - `Дружелюбный, на "ты" (рекомендуется)`
  - `Более формальный, на "вы"`
- Обязательный: Да

---

### Раздел 7: Дополнительно

**Вопрос 7.1**
- Тип: Длинный текст
- Вопрос: `Комментарии или пожелания`
- Описание: `Если есть особые пожелания к настройке бота`
- Обязательный: Нет

**Вопрос 7.2**
- Тип: Множественный выбор (чекбоксы)

  - `Я прочитал инструкцию по подготовке данных`
- Обязательный: Да (все галочки)

## Настройки формы

### Общие настройки
- Собирать email: Да
- Ограничить 1 ответом: Нет (партнёр может заполнить повторно при ошибке)
- Показывать прогресс: Да
- Перемешивать вопросы: Нет

### Уведомления
- Включить уведомления на email при новом ответе

### Ответы
- Сохранять в Google Таблицу для удобства обработки

---

## После создания формы

1. Скопируй ссылку на форму
2. Добавь ссылку в инструкцию для партнёров (PARTNER_ONBOARDING_GUIDE.md)
3. Протестируй форму сам

---

## Обработка ответов

Когда приходит новый ответ:

1. Проверь данные на корректность
2. Скачай JSON-файл с ключом
3. Запусти скрипт:
   ```bash
   python scripts/create_partner.py --interactive
   ```
   или создай config.yaml вручную

4. Разверни бота на сервере
5. Напиши партнёру, что бот готов
